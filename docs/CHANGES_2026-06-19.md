## 5. Data correctness (counts mismatch + "closed twice")
**Why 54 / 60 / 70 disagreed:** three different "open" definitions and two
collapse rules across the code, plus dept-scoping/joint-ticket fan-out. Concrete
fixes applied:
- **Dashboard open count** now collapses the Master Log with **last-non-empty
  wins** (matching the trackers). Previously absolute last-write-wins let a
  trailing blank-status row overwrite a real status with `''`, silently dropping
  open tickets (`worker.js` `handleDashboardCounts`, `Dashboard.gs`).
- **Ticket-detail history** is now **sorted chronologically** by timestamp; raw
  append order mixed backdated imports and produced the "impossible
  back-to-back" / "closed twice" display (`worker.js`, `TicketQueries.gs`).
- **Izzy import contradiction guard** (`IzzySync.gs`): the per-column
  last-non-empty merge could leave a stale `DATE_CLOSED` / `VERIFIED` stamp on a
  ticket Izzy reopened, writing a row that was simultaneously OPEN and "closed on
  date X" (e.g. MT-004-260608-007). The import now clears close/verify fields
  that don't belong to the resolved status.

> Remaining count differences vs Izzy's sheet are definitional (her status
> vocabulary + joint-ticket fan-out across trackers), documented in the audit.
